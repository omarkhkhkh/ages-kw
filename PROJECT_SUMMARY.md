# ملخص مشروع "سجل المناقصات"

نظام إدارة المناقصات الحكومية لشركات المقاولات — يتتبع دورة حياة المناقصة من الترسية وحتى الإغلاق، مع وحدات مساندة للعقود والموردين والمالية والنقل.

- **إجمالي الـ commits**: 79 (من 2026-06-25 حتى 2026-07-08)
- **الحالة**: تطبيق شبه مكتمل، متعدد الوحدات، قيد التطوير النشط

## البنية التقنية (Stack)

| الطبقة | التقنية |
|---|---|
| Monorepo | pnpm workspaces |
| Backend | Express 5 (Node.js 24, TypeScript 5.9) |
| Database | PostgreSQL + Drizzle ORM |
| Validation | Zod (`zod/v4`) + `drizzle-zod` |
| API codegen | Orval — يولّد الـ hooks وschemas من OpenAPI spec |
| Frontend | React + Vite، TanStack Query، واجهة RTL بالكامل (عربي) |
| Build | esbuild (CJS bundle) |

بنية المجلدات الرئيسية:
- `lib/db/src/schema/` — تعريفات جداول قاعدة البيانات
- `lib/api-spec/openapi.yaml` — مصدر الحقيقة لواجهة الـ API
- `artifacts/api-server/src/routes/` — كل الـ endpoints
- `artifacts/tender-manager/src/pages/` — كل صفحات الواجهة

## الوحدات (Modules) اللي اتعملت

كل وحدة فيها: جدول DB + API route + صفحة واجهة (غالبًا بفلترة وترتيب وصفحات pagination).

1. **المناقصات (Tenders)** — القلب الرئيسي: لوحة تحكم، قائمة بفلاتر وتبويبات حالة، نموذج إنشاء، صفحة تفاصيل (عرض/تعديل/حذف)، صلاحيات لكل مناقصة (`tender-permissions`)
2. **العقود (Contracts)** — إدارة وعرض العقود مرتبطة بالمناقصات
3. **الضمانات البنكية (Bank Guarantees)** — قائمة، فلترة وترتيب بالحالة
4. **أوامر الشراء المباشر (Direct Purchase Orders)**
5. **طلبات عروض الأسعار (RFQ Requests)**
6. **الموردون (Suppliers)** — قائمة بفلترة
7. **الممارسات (Practices)** — قائمة بفلترة وترتيب
8. **المشاريع (Projects)**
9. **النقل والمواصلات (Transportation)** — مع فرق نقل ومهام نقل (`transport-teams`, `transport-tasks`)
10. **المالية (Finance)**
11. **المستندات والجهات الحكومية** — مستندات الشركة (`company-documents`) والتسجيلات الحكومية (`government-registrations`) والجهات الحكومية (`government-entities`)
12. **المهام (Tasks)** — مع فلترة وترتيب
13. **التقويم (Calendar)** — widget في الداشبورد + صفحة كاملة
14. **سجل النشاط (Activity Log)** — تتبع كل العمليات، بفلترة وترتيب وpagination
15. **استخبارات المنافسين (Competitor Intelligence)** — وحدة متقدمة: قائمة منافسين، صفحة تفاصيل منافس، تحليلات تنبؤية (`predict`)، نتائج المزايدات (`bid-results`)، لوحة تحليلات (`competitor-analytics`)
16. **الإدارة (Admin)** — إدارة المستخدمين، صلاحيات، أدوار (RBAC)
17. **المصادقة (Auth)** — تسجيل دخول، JWT مع آلية تجديد التوكن (refresh)، middleware للتحقق من الصلاحيات

## أهم القرارات المعمارية

- الواجهة RTL بالكامل (عربي أولاً)
- الـ status enum مطبّق في السيرفر (validation) والعميل (constants map) مع بعض
- منطق "urgency" (المناقصات المستعجلة) بيستثني الحالات المكتملة/المرسلة، متطابق سيرفر/عميل
- الـ cache بيتعمله invalidate عبر TanStack Query query-key helpers بعد كل mutation
- الصلاحيات مبنية على الأدوار (role-based) مع middleware مخصص للتحقق من صلاحيات كل مناقصة
